Snap Memories
Processor

Récupérez vos souvenirs Snapchat exportés avec leurs overlays, horodatages et localisations — en quelques commandes.

v1.0.0 Windows · macOS · Linux ✓ Open source · MIT

Comment ça fonctionne ?

Suivez ces étapes pour récupérer vos souvenirs Snapchat complets.

1

Exporter vos données depuis Snapchat

Connectez-vous à votre compte Snapchat (téléphone ou PC) et rendez-vous sur la page Mes données :

🔗 Ouvrir « Mes données » sur Snapchat

Puis allez dans Paramètres → Mes données et cochez ces deux cases :

  • Exporter mes Souvenirs
  • Exporter des fichiers JSON
⚠️ Les fichiers JSON sont indispensables pour restaurer les métadonnées. Sans eux, l'outil ne peut pas fonctionner.

Choisissez la plage de dates souhaitée, puis patientez. Snapchat vous enverra un e-mail quand l'export sera prêt.

2

Télécharger les archives ZIP

Sur la même page Mes données, dépliez la section « Vos exports ». Vous verrez plusieurs archives ZIP.

ℹ️ Téléchargez toutes les archives ZIP, pas seulement la première. Elles peuvent être réparties si vous avez beaucoup de souvenirs.
3

Organiser les fichiers

Placez tous les fichiers ZIP dans un même dossier. Les archives peuvent porter n'importe quel nom et se trouver n'importe où sur votre système.

Exemple de structure
exports/
├── mydata.zip
├── mydata-2.zip
├── mydata-3.zip
└── mydata-4.zip
4

Vérifier les prérequis

Avant de lancer le traitement, assurez-vous que FFmpeg est bien installé :

Vérification 
ffmpeg -version

Sur macOS et Linux, rendez aussi l'exécutable smp exécutable :

macOS / Linux 
chmod +x smp
💡 Sur Windows, aucune étape supplémentaire n'est nécessaire : double-cliquez sur le .exe ou lancez-le depuis un terminal.
5

Lancer le traitement

Ouvrez un terminal dans le dossier contenant l'exécutable smp, puis lancez :

Terminal 
smp process -i ./exports

Sur macOS/Linux, si smp n'est pas dans le PATH :

macOS / Linux 
./smp process -i ./exports

Une barre de progression s'affiche. Patientez jusqu'à la fin.

6

Récupérer les fichiers traités

Les médias sont enregistrés dans le dossier output/ créé automatiquement à côté de l'exécutable.

Résultat attendu

ℹ️ En cas d'erreurs, un fichier output/errors.log est généré automatiquement avec les détails de chaque échec.

1. FFmpeg Requis

Nécessaire pour traiter les vidéos et les overlays. Sans FFmpeg, l'outil ne peut pas fonctionner.

PowerShell
winget install ffmpeg
⚠️Si winget n'est pas reconnu, installez App Installer depuis le Microsoft Store.
Terminal
brew install ffmpeg
Ubuntu / Debian
sudo apt install ffmpeg
Arch Linux
sudo pacman -S ffmpeg

Vérifiez l'installation :

Vérification
ffmpeg -version

2. ExifTool Optionnel — GPS

Nécessaire uniquement pour conserver les données GPS dans vos médias (flag --gps).

PowerShell
winget install -e --id OliverBetz.ExifTool
Terminal
brew install exiftool
Ubuntu / Debian
sudo apt install libimage-exiftool-perl
Arch Linux
sudo pacman -S perl-image-exiftool
Vérification
exiftool -ver

Questions fréquentes

Les réponses aux problèmes les plus courants.

Pourquoi dois-je cocher « Exporter les fichiers JSON » ?
Les fichiers JSON contiennent les métadonnées de chaque souvenir : date, heure, localisation GPS et informations sur l'overlay. Sans ces fichiers, l'outil n'a aucun moyen de savoir quels overlays appliquer ni comment horodater vos médias.
Le traitement est-il lent ? Comment l'accélérer ?
La durée dépend du nombre de médias et de votre machine. Augmentez le nombre de workers avec -w (ex : -w 8) pour traiter plus de fichiers en parallèle. La valeur par défaut est généralement suffisante.
Que se passe-t-il si certains fichiers échouent ?
Un fichier output/errors.log est créé automatiquement avec les détails de chaque échec. Les fichiers traités avec succès sont disponibles normalement dans output/.
Mes données restent-elles privées ?
Oui. L'outil fonctionne entièrement en local sur votre machine. Aucune donnée n'est envoyée vers des serveurs distants. Vos souvenirs ne quittent jamais votre ordinateur.
J'ai l'erreur « smp : command not found » — que faire ?
Lancez le terminal dans le dossier où se trouve l'exécutable smp. Sous Linux/macOS, rendez-le exécutable avec chmod +x smp puis appelez-le avec ./smp process -i ./exports.
Le flag --gps est activé mais les coordonnées n'apparaissent pas
Vérifiez qu'ExifTool est installé avec exiftool -ver. Notez aussi que tous les souvenirs ne contiennent pas de données GPS — seuls ceux enregistrés avec la localisation active en auront.

Absolument ! Le projet est open source sous licence MIT. Forkez le dépôt et ouvrez une Pull Request sur GitHub.
